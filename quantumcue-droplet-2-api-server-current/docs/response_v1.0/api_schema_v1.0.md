# ML Metrics & Curves API — Schema Documentation

**Version:** 1.0  
**Last Updated:** 2026-02-19  
**Transport:** FastAPI / REST  
**Format:** JSON  

---

## Overview

This API returns synthetic (debug) and real ML evaluation data for multi-class binary
classification models. Each response is identified by a unique `job_id` and contains:

- **Scalar summary metrics** per class (Accuracy, Precision, AUC-ROC, AUC-PR, Max F1)
- **Curve data** per class (F1 vs Threshold, ROC, PR, Confusion Rates vs Threshold)

Responses are generated by the following Python modules (located in `modules/`):

| Module | Responsibility |
|---|---|
| `metrics_generator.py` | Accuracy, Precision, AUC-ROC, AUC-PR, Max F1 scalar arrays |
| `f1_generator.py` | F1 score vs. threshold curves |
| `roc_pr_generator.py` | ROC and PR curves |
| `confusion_generator.py` | TPR, FPR, TNR, FNR vs. threshold curves |

## Field Reference

### Root Fields

| Field | Type | Description |
|---|---|---|
| `job_id` | `string` | Unique job identifier. Format: `job_YYYYMMDD_<hash>` |
| `created_at` | `string` | ISO 8601 UTC timestamp of data generation |
| `num_classes` | `int` | Number of classes in the classification task |
| `num_points` | `int` | Number of points in each curve array |

---

### `metrics` Object

One scalar value per class. All arrays have length = `num_classes`.  
All values are in the range **[0.80, 0.95]**.

| Field | Type | Description |
|---|---|---|
| `accuracy` | `float[]` | Overall classification accuracy |
| `precision` | `float[]` | Macro-averaged precision score |
| `auc_roc` | `float[]` | Area under the ROC curve |
| `auc_pr` | `float[]` | Area under the Precision-Recall curve |
| `max_f1` | `float[]` | Maximum F1 score over all thresholds |

---

### `curves` Object

#### `curves.f1`

F1 score as a function of classification threshold.  
Shape of `scores`: `[num_classes][num_points]`.

| Field | Type | Description |
|---|---|---|
| `thresholds` | `float[]` | Shared x-axis. Length = `num_points`. Range: `[0.0, 1.0]` |
| `scores` | `float[][]` | F1 values. Shape = `[num_classes][num_points]` |

**Curve behavior:**  
Starts ~0.10 at threshold=0, peaks ~0.75 at threshold≈0.45, returns to ~0.20 at threshold=1.

---

#### `curves.roc`

Array of ROC curve objects, one per class. Length = `num_classes`.  
Each class has its own independent `fpr` x-axis.

| Field | Type | Description |
|---|---|---|
| `fpr` | `float[]` | False Positive Rate. Length = `num_points`. Range: `[0.0, 1.0]` |
| `tpr` | `float[]` | True Positive Rate. Length = `num_points`. Range: `[0.0, 1.0]` |

**Curve behavior:**  
Starts at (0, 0), rises steeply near FPR=0, saturates near TPR=1.  
Passes approximately through (FPR=0.10, TPR=0.94). Ends at (1, 1).  
No inflection points. Shape: `tpr = (1+a)*fpr / (fpr + a)`.

---

#### `curves.pr`

Array of Precision-Recall curve objects, one per class. Length = `num_classes`.  
Each class has its own independent `recall` x-axis.

| Field | Type | Description |
|---|---|---|
| `recall` | `float[]` | Recall values. Length = `num_points`. Range: `[0.0, 1.0]` |
| `precision` | `float[]` | Precision values. Length = `num_points`. Range: `[0.0, 1.0]` |

**Curve behavior:**  
Starts at (0, 1.0), stays high for most of the recall range, drops sharply near recall=1.0.  
Ends at (1, 0). Monotone decreasing. No inflection points.

---

#### `curves.confusion_rates`

TPR, FPR, TNR, FNR as functions of threshold.  
All matrices have shape `[num_classes][num_points]`.  
All classes share the same `thresholds` x-axis.

| Field | Type | Description |
|---|---|---|
| `thresholds` | `float[]` | Shared x-axis. Length = `num_points`. Range: `[0.0, 1.0]` |
| `tpr` | `float[][]` | True Positive Rate. Shape = `[num_classes][num_points]` |
| `fpr` | `float[][]` | False Positive Rate. Shape = `[num_classes][num_points]` |
| `tnr` | `float[][]` | True Negative Rate = `1 - FPR`. Shape = `[num_classes][num_points]` |
| `fnr` | `float[][]` | False Negative Rate = `1 - TPR`. Shape = `[num_classes][num_points]` |

**Curve behavior:**  
- `tpr`: Decreasing sigmoid from ~1.0 (threshold=0) to ~0.3 (threshold=1)  
- `fpr`: Decreasing, always strictly below `tpr`. Max at threshold=0  
- `tnr`: Mirror of `fpr` (increasing)  
- `fnr`: Mirror of `tpr` (increasing)  

---

## Design Decisions

| Decision | Rationale |
|---|---|
| `roc` and `pr` are arrays of objects | Each class has an independent x-axis (`fpr`/`recall`), not shared |
| `f1` and `confusion_rates` use shared `thresholds` + 2D matrix | Same x-axis across classes → compact layout, easier client-side plotting |
| Max nesting depth = 3 | Keeps parsing simple for frontend/client engineers |
| Floats rounded to 4 decimal places | Reduces JSON payload size without meaningful precision loss |
| `job_id` as root-level identifier | Enables caching, deduplication, and API logging on DigitalOcean |
| ISO 8601 UTC for `created_at` | Timezone-safe for distributed deployments |

## Python Generator Parameters

Default values used when calling from `main.py`:

| Parameter | Default | Description |
|---|---|---|
| `num_classes` | `4` | Number of output classes |
| `num_points` | `50` | Points per curve (ROC/PR); `20` for F1/confusion |

## Notes for Engineers

- **Debug mode**: All data is synthetically generated. Replace module calls with real  
  model outputs when moving to production. Function signatures remain identical.
- **Array serialization**: Convert `np.ndarray` to `list` via `.tolist()` before  
  `json.dumps()` or FastAPI `JSONResponse`.
- **ROC array of objects vs matrix**: Do not flatten `roc`/`pr` into matrices —  
  each class may eventually have different-length arrays in production.
- **Versioning**: Increment the version field at top of this document on any  
  breaking schema change and notify client team before deployment.
- **DigitalOcean deployment**: `job_id` should be generated server-side using  
  `uuid` or a timestamp hash at request time, not by the client.
